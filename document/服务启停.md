# AgentForge 服务启停命令

默认端口：
- 后端 API：`8000`
- 前端开发：`5173`

---

## 一、启动

### 后端（FastAPI + Uvicorn）

在项目根目录执行：

```bash
bash scripts/run_api.sh
```

或手动启动：

```bash
cd ~/python/AgentForge
.venv/bin/uvicorn agentforge.api.main:app --host 127.0.0.1 --port 8000 --reload
```

访问：
- API 文档：http://127.0.0.1:8000/docs
- 健康检查：http://127.0.0.1:8000/api/health

### 前端（Vue 3 + Vite）

新开一个终端：

```bash
cd ~/python/AgentForge/frontend
npm run dev
```

访问：http://127.0.0.1:5173

对话编程多轮场景示例见：[对话编程场景示例.md](./对话编程场景示例.md)

## 二、停止 / 杀进程

### 按端口杀（推荐）

```bash
# 杀后端（8000）
lsof -ti :8000 | xargs kill -9

# 杀前端（5173）
lsof -ti :5173 | xargs kill -9
```

### 按进程名杀

```bash
# 杀 uvicorn
pkill -f "uvicorn agentforge.api.main:app"

# 杀 vite 开发服务
pkill -f "vite"
```

### 优雅停止

在启动服务的终端里按 `Ctrl + C` 即可。

## 五、常见问题

### 端口已被占用（address already in use）

```bash
lsof -ti :8000 | xargs kill -9   # 或 5173
```

然后重新启动服务。

### 后端启动失败：环境变量未配置

确认已设置：
- `POSTGRESQL_USER` / `POSTGRESQL_PASSWORD`
- `QWEN_API_KEY` / `QWEN_BASE_URL`

### 前端无法连后端

1. 确认后端已在 `8000` 端口运行
2. 前端开发模式通过 Vite 代理 `/api` → `http://127.0.0.1:8000`，无需额外配置

---

## 六、推荐启动顺序

1. 确认 PostgreSQL、Redis 已运行
2. 启动后端：`bash scripts/run_api.sh`
3. 启动前端：`cd frontend && npm run dev`
4. 浏览器打开 http://127.0.0.1:5173

---

## 七、日志（loguru）

- 控制台：启动后端时直接输出
- 文件：`logs/agentforge.log`（10MB 轮转，保留 14 天）
- 日志级别：环境变量 `LOG_LEVEL`（默认 `INFO`）

```bash
# 实时查看日志
tail -f logs/agentforge.log

# 调试模式启动
LOG_LEVEL=DEBUG bash scripts/run_api.sh
```
