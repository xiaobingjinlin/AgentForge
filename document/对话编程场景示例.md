# AgentForge 对话编程场景示例

本文档基于当前已实现能力，提供一套**可复现的多轮对话场景**：从零开始，通过自然语言逐步叠加技术栈并生成业务模块。

适用环境：Spring Boot 4.0、本地 PostgreSQL / Redis、Qwen API 已配置（环境变量见 [服务启停.md](./服务启停.md)）。

---

## 一、场景目标

用对话完成一个 **Order 订单模块** 的后端工程，经历三个阶段：

| 阶段 | 对话意图 | 系统能力 |
|------|----------|----------|
| 1 | 接入 API 文档 | Phase 1/2：启用官方 `springdoc` 能力层 |
| 2 | 接入数据访问 | Phase 3：LLM 自动造 `mybatis` 层（若 Registry 无） |
| 3 | 生成业务代码 | 三层 Agent：Router → 域 SubGraph → Integrator |
| 4 | 落盘与验证 | 一键导出沙盒、本地启动验证 |

最终能力栈示意：

```
base + springdoc + mybatis  →  生成 Order Entity/Mapper/Service/Controller
```

## 二、启动服务

```bash
# 终端 1：后端
bash scripts/run_api.sh

# 终端 2：前端
cd frontend && npm run dev
```

浏览器打开：http://127.0.0.1:5173

## 三、多轮对话脚本

以下按推荐顺序发送。每轮说明：**你说什么 → 系统做什么 → 你如何确认**。

---

### 第 1 轮：叠加 SpringDoc（官方能力层）

**你说：**

```
给项目加上 springdoc，我需要 Swagger 接口文档
```

**系统行为：**

1. `CapabilityRouter` 识别已注册能力 `springdoc`
2. 自动合并 `pom` 依赖、`OpenApiConfig.java`、`application.yml` 配置
3. 更新项目 `template_stack` → `base + springdoc`
4. 返回说明（含 Swagger 访问地址）

**界面提示：**

- 状态栏：`已自动启用能力层: springdoc`
- 能力栈：`base + springdoc`

**你可确认：**

- 右侧无业务代码预览（本轮仅为能力叠加）
- 助手回复含：`http://localhost:8080/swagger-ui.html`

**等价 API（可选）：**

```bash
curl -X POST http://127.0.0.1:8000/api/projects/{project_id}/capabilities/springdoc \
  -H "Content-Type: application/json" -d '{"verify": false}'
```

---

### 第 2 轮：叠加 MyBatis（LLM 造层，Phase 3）

> 若你尚未手工添加 `capabilities/mybatis/`，Registry 中没有 mybatis，将触发 **LLM 造层**。

**你说：**

```
给项目集成 mybatis 数据访问
```

**系统行为：**

1. 推断缺失能力 `mybatis`
2. SSE 推送 `capability_generating`（「Registry 无 mybatis，LLM 生成能力层...」）
3. LLM 生成 `manifest.json` + `overlay/`（依赖、配置、可选 Config 类）
4. 沙盒执行 `mvn compile` 验证（无 Maven 时降级为结构校验）
5. 晋升至 `templates/spring-boot/4.0/capabilities/_generated/mybatis/`
6. 自动启用，`template_stack` → `base + springdoc + mybatis`

**界面提示：**

- `LLM 已生成并启用: mybatis`
- 能力栈：`base + springdoc + mybatis`

**你可确认：**

```bash
ls templates/spring-boot/4.0/capabilities/_generated/mybatis/
# 应含 manifest.json 与 overlay/
```

**说明：** 造层只需发生一次；后续项目或对话会直接复用 `_generated/mybatis`，不再调用 LLM。

---

### 第 3 轮：生成 Order 模块 CRUD（核心业务）

**你说：**

```
生成 Order 模块完整 CRUD：Entity、Mapper、Service、Controller
```

**系统行为：**

1. 能力栈已含 springdoc + mybatis，沙盒按栈组合模板后生成代码
2. **RouterAgent** 路由至：`entity → mapper → service → controller`
3. 各域 **SubGraph** 三阶段生成：骨架 → 实现 → 测试修复
4. 每域检索 RAG 知识库（top-k + rerank）作为参考
5. **IntegratorAgent** 流式输出整合方案
6. 代码写入 `sandbox/{project_id}/`

**界面提示：**

- 右侧 **代码预览** 面板按域切换：`entity` / `mapper` / `service` / `controller`
- 底部：`沙盒 N 个文件 · 栈: base + springdoc + mybatis`

**你可确认：**

| 预期文件 | 路径示例 |
|----------|----------|
| 实体 | `src/main/java/com/example/demo/entity/Order.java` |
| Mapper | `src/main/java/com/example/demo/mapper/OrderMapper.java` |
| Service | `src/main/java/com/example/demo/service/OrderService.java` |
| Controller | `src/main/java/com/example/demo/controller/OrderController.java` |

---

### 第 4 轮：一键落盘（可选）

**操作：** 点击界面 **「一键落盘」** 按钮。

**前提：** 创建项目时设置了 `root_path`；否则落盘到沙盒目录并提示路径。

**你说（也可对话追问）：**

```
把当前沙盒代码同步到本地工程
```

**系统行为：** `POST /api/projects/{id}/export` 将沙盒文件复制到 `root_path`。

---

### 第 5 轮：增量完善（对话式迭代）

**你说：**

```
给 OrderController 增加按状态筛选的查询接口 GET /api/orders?status=xxx
```

**系统行为：** 再次走路由 → 主要命中 `controller` 域（可能连带 `service`），在已有沙盒工程上增量生成并落盘。

## 四、单轮组合对话（进阶）

可将多步合并为一条消息：

```
加 springdoc 和 mybatis，然后生成 Order 模块完整 CRUD
```

执行顺序：

1. 启用 / 造层 `springdoc`
2. 启用 / 造层 `mybatis`
3. 运行完整代码生成流水线

> 注意：组合对话会多次调用 LLM（造层 + 各域生成 + 整合），耗时与 token 消耗更高。

## 五、背后架构一览

```
用户消息 (Vue SSE)
    │
    ▼
AgentOrchestrator
    ├─ CapabilityRouter     ← Phase 2/3：识别 & 造层
    ├─ TemplateComposer     ← base + 能力栈 → 沙盒模板
    ├─ LangGraph 主图
    │     ├─ RouterAgent
    │     ├─ Domain SubGraph × N（RAG + 三阶段 codegen）
    │     └─ IntegratorAgent
    └─ 持久化：template_stack / generation_records / file_tree
```

## 六、SSE 事件对照表

| 事件 | 含义 |
|------|------|
| `status` | 阶段提示（capabilities / routing / executing / integrating） |
| `capability_generating` | LLM 正在造层 |
| `capability_generated` | 造层完成并入库 `_generated/` |
| `capability_enabled` | 能力已写入项目栈与沙盒 |
| `domain_result` | 某技术域代码预览 |
| `token` | 整合层流式文本 |
| `done` | 本轮结束 |

## 八、常见问题

**Q：说了「加 react」会怎样？**  
系统会**直接拒绝**：AgentForge 仅支持 Spring Boot 后端生态能力层（如 springdoc、mybatis、redis、kafka）。React/Vue/Django/Go 等不属于本模板范围，不会造层、不会改 `template_stack`，也不会继续走代码生成。助手会返回拒绝说明与可尝试的 Spring Boot 能力示例。

**Q：说了「加 redis」但没反应？**  
需包含启用意图词（如「加」「启用」「集成」），且消息未被判定为纯 CRUD 生成。

**Q：mybatis 造层失败？**  
查看后端日志；可改用 API 单独造层并设 `"verify": false` 跳过编译。

**Q：重复说「加 springdoc」？**  
系统幂等回复「已在项目中」，不会重复写入。

**Q：想改用 Spring Boot 3.2？**  
创建项目时指定 `framework_version: "3.2"`，RAG 与知识库按版本检索。
